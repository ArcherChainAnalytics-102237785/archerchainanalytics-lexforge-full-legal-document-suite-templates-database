- `[PHONE_NUMBER]` — Phone contact

- `[REGISTRATION_NUMBER]` — Corporate/business registration

**Product Variables:**
- `[PRODUCT_NAME]` — Name of product/service
- `[SERVICE_NAME]` — Service description
- `[PRODUCT_DESCRIPTION]` — Full description
- `[PRODUCT_PRICING]` — Price/pricing model
- `[LICENSE_TYPE]` — Perpetual, subscription, tiered, etc.

**Jurisdiction Variables:**
- `[PRIMARY_JURISDICTION]` — Home jurisdiction
- `[GOVERNING_LAW_JURISDICTION]` — Choice of law
- `[APPLICABLE_PROVINCES]` — For Canada (AB, BC, etc.)
- `[APPLICABLE_STATES]` — For USA (CA, NY, etc.)
- `[NEXUS_JURISDICTIONS]` — Where you have economic nexus

**Operational Variables:**
- `[UPTIME_PERCENTAGE]` — SLA (99.9%, 99.99%)
- `[INCIDENT_RESPONSE_TIME]` — Hours for support response
- `[MAINTENANCE_WINDOW]` — Scheduled maintenance hours
- `[DATA_RETENTION_PERIOD]` — How long data is kept

**Compliance Variables:**
- `[WCAG_STANDARD]` — WCAG 2.1 AA, WCAG 2.2 AA, etc.
- `[DATA_BREACH_NOTIFICATION_TIMELINE]` — Days (e.g., 30 days)

## LEXFORGE™ TEMPLATE DATABASE INTEGRATION

### Schema for LexForge™ System

Each document in the LexForge™ database includes:

```json
{
  "document_id": "001_01",
  "doc_name": "Terms of Service",
  "category": "core_compliance",
  "jurisdiction": "canada",
  "language": "en",
  "version": "personal",
  "metadata": {
    "pages": 8,
    "word_count": 2400,
    "last_updated": "2026-06-17",
    "compliance_tags": ["PIPEDA", "CASL", "Consumer Protection Act"],
    "cross_references": ["001_03_privacy_policy", "004_01_limitation_of_liability"],
    "variables": ["COMPANY_NAME", "SERVICE_NAME", "EFFECTIVE_DATE", ...]
  },
  "content_hash": "sha256:...",
  "template_content": "...(full document text)..."
}
```

### Variable Injection System

LexForge™ uses **mustache syntax** for variable injection:

```
{{{COMPANY_NAME}}}
```

When commercial version is used, LexForge™ scans for all `{{{VAR}}}` tags and provides UI for user to input values.
